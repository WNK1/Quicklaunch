## Следующие шаги

1. **Дизайнер** — прислать 5 логотипов заказчику на согласование; приступить к статическим макетам страниц (приоритет: страница кейса → главная → рулетка)
2. **Frontend** — создать HTML-макеты на основе готового CSS: `index.html`, `case.html` (с JS-анимацией спиннера), `roulette.html`, `inventory.html`
3. **Backend** — инициализировать структуру проекта (src/modules/), написать Docker Compose (PostgreSQL + Redis), создать Prisma-схему, реализовать Steam Auth
4. **PM + заказчик** — согласовать логотип, языки сайта, подтвердить наличие Steam API ключа и платёжных аккаунтов, определить бюджет на скины для ботов
5. **Debugger** — ожидать первые HTML-файлы от frontend для начала ревью

## Вопросы к заказчику

| # | Вопрос | Влияние на разработку |
|---|--------|----------------------|
| 1 | Какой вариант логотипа (v1–v5) утверждается? | Финализация брендинга, favicon, OG-изображения |
| 2 | Языки сайта на старте: только RU, RU + EN, или оба сразу? | Трудозатраты верстки, структура i18n, переключатель языков |
| 3 | Есть ли Steam Developer API ключ? | Без него Steam OpenID авторизация невозможна |
| 4 | Созданы ли мерчант-аккаунты Stripe и Cryptomus? | Без них нельзя принимать платежи; Stripe может отказать gambling-проектам |
| 5 | Юридическое лицо и юрисдикция — когда будет создано? | Нужно до открытия платёжных счётов и запуска |
| 6 | Бюджет на скины для Steam Trade Bot-инвентаря? | Определяет сроки подготовки вывода скинов |
| 7 | Кто наполняет каталог кейсов (состав, цены, вероятности) — команда разработки или заказчик? | Влияет на сроки Sprint 2 |
| 8 | Есть ли staging-сервер или VPS для деплоя? Кто управляет DNS? | Нужно для тестирования Steam OAuth (требует публичный URL) |
| 9 | Целевые рынки (СНГ, Европа, весь мир)? | Платёжные системы, geo-blocking, ЮКасса |
